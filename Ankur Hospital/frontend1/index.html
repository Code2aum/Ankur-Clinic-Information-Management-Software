Ankur Hospital

Project By: Harshith R and Harshvardhan H P

Description

Hospitals interact with a lot of people in a day and there are various activities involved in day to day operations of hospitals, for example booking of appointments, managing doctor schedules, managing patient diagnoses, managing medical histories of patients, etc. The aim of this project is to show how data related to these tasks can be made easier to manage using databases.

Patient Side Features

There is a seperate interface for patients. Patients have a seperate login.Patients can book appointments.Patients can give previous medical history. Patients can view/update/cancel already booked appointments if necessary. Cancelled appointments create free slots for other patients. The system avoids clash of appointments with other patients. Each patient is therefore ensured his/her slot. Patients are able to see complete diagnosis, prescriptions and medical history. Patient medical history is only available to the doctor with whom the appointment is booked to ensure privacy.

Doctor Side Features

There is a seperate interface for doctors. Doctors have a seperate login. The system takes into consideration doctor schedules and does not allow appointments when a doctor is already busy or has a break. Doctors are able to access patient history and profile, and add to patient history. Doctors are able to give diagnosis and prescriptions. Doctors are able to modify diagnosis and prescriptions.
